//pacotes
🧩 1. Projeto: xFood.Domain

📂 Contém apenas entidades, nada de EF, nada de dependência externa.

✅ Nenhum pacote necessário

dotnet add package Microsoft.EntityFrameworkCore.Design --version 9.0.0-preview.*


Apenas referências padrão do .NET (System.*, DataAnnotations, etc.).
O SDK já traz tudo o que você precisa.




2. Projeto: xFood.Application

📂 Camada de regras e abstrações (DTOs, Interfaces).

✅ Nenhum pacote obrigatório neste momento.

Se quiser evoluir depois (ex.: AutoMapper, FluentValidation), podemos incluir.
Mas para esta fase simples: zero dependências externas.


🗃️ 3. Projeto: xFood.Infrastructure

📂 Contém DbContext, repositórios e o SeedService.

Pacotes obrigatórios
dotnet add package Microsoft.EntityFrameworkCore --version 9.0.0-preview.*
dotnet add package Microsoft.EntityFrameworkCore.SqlServer --version 9.0.0-preview.*
dotnet add package Microsoft.EntityFrameworkCore.Design --version 9.0.0-preview.*
dotnet add package Microsoft.Extensions.Configuration --version 9.0.0-preview.*
dotnet add package Microsoft.Extensions.Configuration.FileExtensions --version 9.0.0-preview.*
dotnet add package Microsoft.Extensions.Configuration.Json --version 9.0.0-preview.*


🌐 4. Projeto: xFood.Web (MVC + API)

📂 É o frontend MVC e também a API.

Pacotes obrigatórios
dotnet add package Microsoft.EntityFrameworkCore.SqlServer --version 9.0.0-preview.*
dotnet add package Microsoft.EntityFrameworkCore.Tools --version 9.0.0-preview.*
dotnet add package Microsoft.AspNetCore.Mvc.Razor.RuntimeCompilation --version 9.0.0-preview.*
dotnet add package Microsoft.Extensions.Configuration.Json --version 9.0.0-preview.*


O pacote Razor.RuntimeCompilation é opcional, mas ótimo pra estudar —
ele permite editar .cshtml e ver mudanças sem recompilar.

Pacotes opcionais (para futuras aulas)
dotnet add package Swashbuckle.AspNetCore --version 6.6.2     # Swagger
dotnet add package Microsoft.AspNetCore.Authentication.Cookies --version 9.0.0-preview.*
